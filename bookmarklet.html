🏋️ Gym Bot — Come dare i cookie al bot

Perché serve DevTools (e non un bottone magico): i cookie di login del sito (.ASPXAUTH_SOCIALBOOKING, ASP.NET_SessionId, __RequestVerificationToken) sono HttpOnly: per sicurezza il browser non li rende copiabili da uno script. L'unico modo è copiarli dalla scheda Network con «Copy as cURL». Sono 5 click, si fa solo quando il bot dice «sessione scaduta».
[1] Login SPID -> [2] F12 / Network -> [3] Copy as cURL -> [4] /cookies + incolla sul sito apri DevTools tasto destro sul bot Telegram
1Fai login
Apri Chrome o Edge su ecomm.sportrick.com/sportpolimi e accedi con SPID, come fai sempre.
2Apri DevTools → Network
Premi F12 (oppure tasto destro → Ispeziona).
In alto scegli la scheda Network (Rete). Poi ricarica la pagina con F5: comparirà una lista di righe.
3Copia come cURL
Clicca una riga qualsiasi che punta a ecomm.sportrick.com (es. la prima, sportpolimi o Booking).
Tasto destro sulla riga → Copy → Copy as cURL.
(Su Windows va bene sia «cURL (bash)» sia «cURL (cmd)».)
4Mandalo al bot
Apri Telegram, chat col bot, scrivi /cookies + spazio, poi incolla (Ctrl+V) tutto il testo copiato e invia.
Il bot legge da solo i cookie dal comando cURL.
Risposta attesa: ✅ Sessione salvata!
Se il bot dice «mancano cookie importanti»: hai copiato una richiesta senza login (es. un'immagine o un file statico). Torna al punto 3 e copia una richiesta a una pagina del sito (Booking o sportpolimi), non a immagini/CSS/JS.

In alternativa puoi incollare solo la stringa cookie grezza (nome=valore; nome2=valore2): la trovi in DevTools → riga selezionata → Headers → Request Headers → riga Cookie:.

I cookie restano solo tra il tuo browser, Telegram e il tuo bot. Non vengono condivisi con nessun altro.
